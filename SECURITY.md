# Security Policy

## Reporting a vulnerability

Do not open a public issue for suspected security vulnerabilities.

Instead, report them privately to the maintainers at:

- `security@talosialab.com`

Include:

- a clear description of the issue
- affected versions or commit range if known
- reproduction steps or a proof of concept
- the potential impact

## Response expectations

The project will try to:

- acknowledge receipt within 5 business days
- validate the report and assess impact
- coordinate a fix and disclosure timeline when the report is confirmed

## Scope

Security reports are especially helpful for:

- command execution or shell injection risks
- unsafe prompt or file handling
- credential or secret exposure
- unintended access to files, repositories, or local state

If you are unsure whether something is security-sensitive, report it privately first.
